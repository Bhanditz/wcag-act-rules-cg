Auto-WCAG Rules

# Name Description WCAG Criterion
1 ARIA allowed attribute This rule checks that `aria-*` attributes are allowed for the element they are specified on.
  • 4.1.1 Parsing - (Level: A)
  • 4.1.2 Name, Role, Value - (Level: A)
2 Audio-only as a media alternative for text This rule checks `audio` is a media alternative for text on the page.
3 Autocomplete valid This rule checks that the HTML autocomplete attribute has a correct value
  • 1.3.5 Identify Input Purpose - (Level: AA)
4 Buttons have an accessible name Each button element has an accessible name
  • 4.1.2 Name, Role, Value - (Level: A)
5 Form field has accessible name Each form field element has an accessible name
  • 3.3.2 Labels or Instructions - (Level: A)
6 HTML Page has a title This rule checks that the HTML page has a title
  • 2.4.2 Page Titled - (Level: A)
7 HTML has lang attribute This rule checks that the `html` element has a non-empty `lang` or `xml:lang` attribute.
  • 3.1.1 Language of Page - (Level: A)
8 HTML lang and xml:lang match The rule checks that for the `html` element, there is no mismatch between the primary language in non-empty `lang` and `xml:lang` attributes, if both are used.
  • 3.1.1 Language of Page - (Level: A)
9 Image has accessible name Each image that is not marked as decorative, has an accessible name
  • 1.1.1 Non-text Content - (Level: A)
10 Links have an accessible name Each link has an accessible name
  • 4.1.2 Name, Role, Value - (Level: A)
  • 2.4.4 Link Purpose (In Context) - (Level: A)
11 Meta-refresh no delay This rule checks that the meta element is not used for delayed redirecting or refreshing.
  • 2.2.1 Timing Adjustable - (Level: A)
  • 2.2.4 Interruptions - (Level: AAA)
  • 3.2.5 Change on Request - (Level: AAA)
12 No keyboard trap This rule checks for keyboard traps. This includes use of both standard and non-standard keyboard navigation to navigate through all content without becoming trapped.
  • 2.1.2 No Keyboard Trap - (Level: A)
13 No keyboard trap non-standard navigation This rule checks if it is possible to use non-standard keyboard navigation to navigate through content where focus is trapped when using standard ways of keyboard navigation.
14 No keyboard trap standard navigation This rule checks if it is possible to use standard keyboard navigation to navigate through all content on a web page without becoming trapped in any element.
15 Role attribute has valid value This rule checks that each role attribute has a valid value
  • 4.1.2 Name, Role, Value - (Level: A)
16 Role has required states and properties Elements that have an explicit role must also specify all required states and properties
  • 4.1.2 Name, Role, Value - (Level: A)
17 Valid body lang attribute This rule checks that `lang` or `xml:lang` attributes on elements within the `body` of a web page has a valid language subtag.
  • 3.1.2 Language of Parts - (Level: AA)
18 Validity of HTML Lang attribute This rule checks the lang or xml:lang attribute has a valid language subtag.
  • 3.1.1 Language of Page - (Level: A)
19 Video element description track This rule checks that description tracks that come with non-streaming `video` elements are descriptive.
20 Video-only as a media alternative for text This rule checks non-streaming silent `video` is a media alternative for text on the page.
21 Vídeo has audio alternative This rule checks that video elements have an alternative for information conveyed through audio
  • 1.2.2 Captions (Prerecorded) - (Level: A)
22 aria-hidden with focusable content This rule checks `aria-hidden` elements do not contain focusable elements
  • 4.1.2 Name, Role, Value - (Level: A)
23 audio described video element Non-streaming `video` elements must have all visual information also contained in the audio
24 audio elements captions Non-streaming `audio` elements must have a text alternative or captions for all included auditory information.
  • 1.2.2 Captions (Prerecorded) - (Level: A)
25 iframe has an accessible name Each iframe element has an accessible name
  • 4.1.2 Name, Role, Value - (Level: A)
26 label and name from content mismatch Interactive elements labelled through their content must have their visible label as part of their accessible name
  • 2.5.3 Label in Name - (Level: A)
27 unique element id attributes This rule checks that all `id` attribute values on a single page are unique.
  • 4.1.1 Parsing - (Level: A)
28 video element transcript Non-streaming `video` elements must have all audio and visual information available in a transcript.
  • 1.2.8 Media Alternative (Prerecorded) - (Level: AAA)
29 video has captions Captions are available for audio information in non-streaming `video` elements.
  • 1.2.2 Captions (Prerecorded) - (Level: A)

Contribute






GitHub

Contributing is open to anyone. We welcome any new issues or pull requests for changes. Auto-WCAG has conference calls every 4 weeks. If you are interested in becoming an active contributor or reviewer, we ask that you join the Auto-WCAG community group through the W3C Website. This requires setting up a W3C account, may require approval by the organization you work for if they are a W3C member.
